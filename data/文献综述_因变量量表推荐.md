## 📊 推荐测量量表及题项

### **一、操纵检验（Manipulation Check）**
**目的**: 验证被试是否感知到四级记忆架构的差异

#### 理论基础与来源

本操纵检验基于**感知个性化（Perceived Personalization）**理论，改编自：

1. **Kalyanaraman & Sundar (2006)** - 感知个性化量表
   - 原始2题测量用户对内容个性化的感知
   - 来源: *Journal of Communication*, 56(1), 110-132
   - 链接: https://onlinelibrary.wiley.com/doi/abs/10.1111/j.1460-2466.2006.00006.x

2. **Aguirre et al. (2015)** - 个性化悖论研究
   - 定义: "用户感知到信息针对自己作为独特个体的程度"
   - 来源: *Journal of Retailing*, 91(1), 34-49
   - 链接: https://ideas.repec.org/a/eee/jouret/v91y2015i1p34-49.html

**改编说明**: 由于现有文献缺乏专门测量"AI记忆能力感知"的量表，本研究将个性化感知扩展至记忆感知，增加3题记忆能力题项，形成5题综合量表。

#### 测量题项

| 维度 | 题项 | 量表 |
|------|------|------|
| 感知记忆能力 | 1. 这个AI能够记住我之前说过的内容 | 1-7 Likert |
| | 2. 这个AI在对话中展现出对我过往信息的记忆 | 1-7 Likert |
| | 3. 这个AI能够回忆起我们之前对话的细节 | 1-7 Likert |
| 感知个性化 | 4. 这个AI的回复是基于对我个人信息的了解 | 1-7 Likert |
| | 5. 这个AI的回应针对我作为独特的个体<br>*(改编自Kalyanaraman & Sundar, 2006)* | 1-7 Likert |

#### 测量时机 ⏰

根据[操纵检验方法学原则](https://pmc.ncbi.nlm.nih.gov/articles/PMC7954782/)，建议采用**重复测量设计**：

| 任务阶段 | 是否测量 | 测量理由 |
|---------|---------|---------|
| **任务1后** | ⚠️ 可选 | 尚无历史可回忆，L1-L4差异不明显 |
| **任务2后** | ✅ **必须** | 首次记忆触发，验证操纵是否成功 |
| **任务3后** | ✅ **建议** | 检验记忆效果持续性（跨时间检验） |
| **任务4后** | ✅ 建议 | 最终验证，检查操纵效果是否衰减 |

**最简方案**: 如担心问卷疲劳，**至少在任务2后测量**（首次记忆触发时）

#### 预期结果与统计检验

**预期模式**（如操纵成功）:
```
L1（感觉记忆）: M = 1-2分（无记忆）
L2（工作记忆）: M = 3-4分（记得近期）
L3（要义记忆）: M = 4-5分（记得大意）
L4（混合记忆）: M = 5-7分（记得详细）
```

**统计检验**:
```python
# One-Way ANOVA
操纵检验总分 ~ 记忆组别(L1/L2/L3/L4)

# 假设检验
H0: μ_L1 = μ_L2 = μ_L3 = μ_L4
H1: 至少有一组均值不同

# 期望结果
F统计量显著 (p < .001)
事后检验: L4 > L3 > L2 > L1 (Tukey HSD)

# 效应量
期望 η² > 0.14 (大效应)
```

**失败标准**: 如果四组间无显著差异（p > .05），则表明被试未能感知到记忆差异，需重新设计操纵方式。

#### 2. 恐怖谷效应（Uncanny Valley）
**量表**: Ho & MacDorman (2017) - 诡异感量表

| 题项（语义差异量表） | -3 ← 量表 → +3 |

## 🔬 数据分析建议

### 1. 信度检验
```python
# 每个量表计算Cronbach's α
# 目标: α > 0.70（可接受），α > 0.80（良好）
```

### 2. 效度检验
- **操纵检验**: ANOVA比较四组记忆水平的感知记忆能力差异
- **区分效度**: 各因变量间相关系数 < 0.85
- **聚合效度**: 同一量表题项间相关 > 0.50

### 3. 主效应分析
```
因变量 ~ 记忆水平 (L1/L2/L3/L4)
使用 One-Way ANOVA + Post-hoc (Tukey HSD)
```
